# Sovereign AI Benchmark Methodology (v0.1.0a2)

## Goal
To provide a transparent, reproducible framework for measuring the effectiveness of the **Verified Airlock** and the performance of the **Forensic Audit Chain**.

## 🔬 Metric 1: Hallucination Blocking Rate (HBR)
We measure the precision of the NLI-based grounding gate using an adversarial dataset.

- **Test Condition**: A query is paired with a correct context block and a manually injected hallucinated response.
- **Success Criteria**: The Airlock must identify the response as `not safe` (Score < Threshold).
- **Target**: HBR ≥ 95% on the `benchmark/dataset.json` reference set.

## ⚡ Metric 2: Forensic Latency Overhead
We measure the wall-clock time added by the cryptographic signing and hashing process.

- **Baseline**: RAG retrieval + LLM generation (Airlock disabled).
- **Hardened**: RAG retrieval + LLM generation + Airlock + TPM Signing.
- **Target**: Overhead < 250ms per request on standard hardware (TPM 2.0).

## 🛡️ Reproducibility
To reproduce these results:
1. Ensure `sovereign-ai-stack` is installed.
2. Run `python -m benchmark.harness --dataset benchmark/dataset.json`.
3. Report the `[Environment]` block generated by `benchmark.py` alongside your results.

## 🔒 Transparency Note
These benchmarks currently focus on **Logical Integrity**. They do not yet account for hardware-level side-channel attacks or cold-boot vulnerabilities in the TPM, which are out of scope for this Alpha Research Preview.
